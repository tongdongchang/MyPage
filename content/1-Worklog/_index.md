**On this page**, you will find an overview of my internship worklog. The entire project was carried out over **8 weeks** (from March 9 to May 1, 2026). During this period, I progressively built a complete **music streaming web application** using React for the frontend, Django for the backend, and AWS for cloud deployment. Each week focused on specific objectives, from learning AWS fundamentals and researching the tech stack to implementing client & admin interfaces, building REST APIs, and finally deploying the application on AWS services like S3, CloudFront, Route 53, and EC2.

Below is a summary of what was accomplished week by week:

**Week 1:** [Getting familiar with AWS and basic AWS services](1.1-week1/) – learned core AWS concepts, created a Free Tier account, explored the Console and CLI, and practiced with EC2.

**Week 2:** [Researching the project and its technology stack](1.2-week2/) – met the team, studied the “Music Streaming Website” requirements, and researched React (frontend) and Django (backend) fundamentals.

**Week 3:** [Building the client-side React UI](1.3-week3/) – developed the main user interface: Home, Register/Login, Album, Playlist, Music Player Bar, Search, and Music Video pages.

**Week 4:** [Building the admin panel React UI](1.4-week4/) – created the administration interface for managing songs, MVs, artists, albums, users, and a dashboard.

**Week 5:** [Building client REST API with Django](1.5-week5/) – implemented backend APIs for the music client: song/artist/album listing, user authentication (JWT), playlist management, and search.

**Week 6:** [Building admin REST API with Django](1.6-week6/) – developed dedicated admin endpoints for full CRUD on all resources, plus statistics for the dashboard.

**Week 7:** [Deploying the application on AWS](1.7-week7/) – deployed the React frontend on S3 + CloudFront + Route 53, and the Django backend on EC2; configured HTTPS and security groups.

**Week 8:** [Final testing, documentation, and project wrap‑up](1.8-week8/) – performed end‑to‑end testing, fixed bugs, wrote technical documentation and user manual, and submitted the final internship report.
